# Changelog

## v2.2 — Safebooru 🟢

- **Новая команда `/safebooru`.** Поиск по тегам на Safebooru — это safe-контент
  (без хентая), поэтому команда работает **в любом канале**, а не только в NSFW.
  До 4 тегов, выдача — самое залайканное по реакциям.
- **Та же система тегов.** Блокировки и «приколдес»-теги (как у `/gelbooru`)
  действуют и здесь, плюс общий анти-повтор из v2.1.

## v2.1 — меньше повторов 🔁

Бот стал заметно реже присылать одно и то же.

- **Память переживает рестарт.** Список уже показанных артов теперь сохраняется на
  диск (`recent_shown.json`) и подгружается при старте — раньше каждый
  перезапуск на хостинге обнулял память, и повторы возвращались с нуля.
- **Помним больше.** На каждый набор тегов держим в памяти последние 60 артов
  вместо 25 — популярные теги дольше не повторяются.
- **Шире выбор.** Среди качественных артов (по реакциям) выбор стал почти
  равномерным: один мега-популярный арт больше не вылезает в большинстве выдач.
  Приоритет лайкам сохраняется — мусор по-прежнему остаётся внизу.

## v2.0 — арты из Telegram и умный поиск 🚀

Большое обновление: бот научился доставать арты прямо из Telegram-каналов, а сам
поиск стал заметно умнее. Шанс наткнуться на рекламу или мусор теперь почти нулевой.

### 🆕 Новая команда `/tg`
- **Поиск артов в Telegram.** Бот тянет картинки и гифки из наших Telegram-каналов и
  присылает их прямо в Discord — больше не нужно никуда переходить.
- **Выбор канала.** `/tg` — случайный канал из списка, либо `/tg channel:<название>` —
  конкретный. С автоподсказкой названий прямо при вводе.

### ✨ Поиск стал умнее
- **Минимум мусора.** Реклама, посты с кнопками-ссылками и записи без картинок
  отсеиваются автоматически — шанс наткнуться на фигню почти нулевой.
- **Сначала лучшее.** Арты ранжируются по реакциям, так что наверх всплывает то, что
  реально зашло аудитории, — но с долей случайности, чтобы не было однообразия.
- **Без повторов.** Бот помнит, что уже показывал, и старается не присылать одно и то же.
- **Только в NSFW.** Команда работает исключительно в NSFW-каналах — как и положено. 🔞

### 🎬 Медиа
- Большие файлы обрабатываются аккуратно: если арт не влезает в лимит Discord, бот
  подберёт следующий подходящий, а не выдаст ошибку.

## v1.5 — исправления и полировка

### 🔒 Безопасность
- Токен бота и `GELBOORU_API_KEY`/`GELBOORU_USER_ID` больше **не зашиты в код** — читаются из
  переменных окружения (`DISCORD_BOT_TOKEN`, `GELBOORU_API_KEY`, `GELBOORU_USER_ID`).
- `bot.run(...)` убран из тела модуля и обёрнут в `if __name__ == "__main__"` — теперь файл
  можно импортировать (например, для тестов) без автоматического подключения к Discord.

### 🔍 Поиск и фильтры
- **Реальная случайность результатов.** Gelbooru игнорирует параметр `sort=random`; теперь
  используется тег `sort:random`. Раньше бот выдавал одни и те же (самые свежие) посты.
- **Фильтр больше не пропускает запрещённый контент.** Убран откат к нефильтрованному списку
  при наличии менее 5 «чистых» постов — отдаются только прошедшие блэклист.
- **Точное совпадение блокированных тегов.** Проверка по подстроке ошибочно блокировала
  легитимные теги (`futaba_sakura`, `strap`, `contrast` из-за `futa`/`trap`). Теперь сравнение
  по токенам тега.

### ⏱️ Кулдауны
- Кулдауны теперь действительно работают. Декоратор `@commands.cooldown` для слэш-команд
  nextcord молча игнорировался; реализован собственный sliding-window кулдаун
  (`/gelbooru` 3/30с, `/tagcheck` 5/30с, `/tags` 1/30с).

### 🎬 Медиа
- **Видео (mp4/webm)** теперь скачивается и отправляется файлом — Discord его проигрывает.
  Раньше отдавалась только ссылка. При превышении лимита — фолбэк на ссылку.
- Лимит размера файла берётся из реального лимита сервера (`guild.filesize_limit`) вместо
  жёстких 8 МБ; для личных сообщений — безопасные 10 МБ.
- Логика скачивания (HEAD-проверка размера + ретраи) вынесена в общий помощник `download_media`.

### 🛠️ Стабильность
- Исправлен жизненный цикл HTTP-сессии: `on_disconnect` больше не закрывает общую сессию при
  кратковременных разрывах, а `on_ready` не создаёт новую сессию при каждом реконнекте (утечка).
- Добавлен глобальный обработчик ошибок команд `on_application_command_error`.
- NSFW-проверка безопасна для веток/тредов и нестандартных каналов (`getattr` вместо прямого вызова).
- Устойчивый парсинг ответа: одиночный пост-словарь и формат `{"tag": [...]}` обрабатываются корректно.
- Единый сборщик параметров запроса `base_params()` (ключи API добавляются только если заданы).

### ✅ Тесты
- Добавлен `test_bot.py` — 29 юнит-тестов (parse_posts, post_is_clean, блок-теги, кулдаун, blacklist, base_params).
